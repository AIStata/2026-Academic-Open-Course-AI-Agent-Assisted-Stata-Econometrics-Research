3  第二讲：Do-file 运行与结果管理

3.1 Do-file 开发实践

3.1.1 代码结构规范与注释标准

推荐的 Do-file 结构

一个结构良好的 do-file 应该包含以下部分：

*-------------------------------------------------------------------------------
*  project_name/analysis/main_analysis.do
*  目的: 主要回归分析
*  作者: 研究者姓名
*  创建日期: 2026-06-24
*  修改记录:
*  - 2026-06-25: 添加稳健性检验
*-------------------------------------------------------------------------------

*-------------------------------------------------------------------------------
*  1. 初始化
*-------------------------------------------------------------------------------

clear all
set more off
set linesize 120
set varabbrev off
set seed 20260624

* 设置全局路径
global root   "D:/research/project_name"
global data   "$root/data"
global raw    "$data/raw"
global clean  "$data/clean"
global results "$root/results"
global logs   "$results/logs"
global figures "$results/figures"
global tables "$results/tables"

* 创建目录（如不存在）
cap mkdir "$results"
cap mkdir "$logs"
cap mkdir "$figures"
cap mkdir "$tables"

* 开始日志记录
log using "$logs/main_analysis_$S_DATE.log", replace text

* 显示开始时间
display "开始时间: $S_TIME $S_DATE"

*-------------------------------------------------------------------------------
*  2. 加载数据
*-------------------------------------------------------------------------------

use "$clean/analysis_data.dta", clear

* 数据检查
isid id
count
summarize

*-------------------------------------------------------------------------------
*  3. 变量处理
*-------------------------------------------------------------------------------

* 创建对数变量
gen ln_price = log(price)
gen ln_income = log(income)

* 创建交互项
gen treat_post = treat * post

* 创建分组变量
egen age_group = cut(age), at(18,30,45,60,100) label

*-------------------------------------------------------------------------------
*  4. 描述性统计
*-------------------------------------------------------------------------------

estpost summarize price mpg weight length foreign if e(sample)
esttab using "$tables/summary_stats.tex", replace ///
    cells("mean(fmt(2)) sd(fmt(2)) min(fmt(2)) max(fmt(2)) count(fmt(0))") ///
    label nonumber nomtitle collabels("均值" "标准差" "最小值" "最大值" "观测数")

*-------------------------------------------------------------------------------
*  5. 回归分析
*-------------------------------------------------------------------------------

estimates clear

* 模型1: 基准回归
regress price mpg weight i.rep78, robust
estimates store m1

* 模型2: 添加控制变量
regress price mpg weight length i.rep78, robust
estimates store m2

* 模型3: 固定效应
reghdfe price mpg weight length, absorb(rep78) vce(robust)
estimates store m3

* 输出回归表格
esttab m1 m2 m3 using "$tables/regression_results.tex", replace ///
    se star(* 0.1 ** 0.05 *** 0.01) ///
    label booktabs alignment(D{.}{.}{-1}) ///
    keep(mpg weight length) ///
    mtitles("模型1" "模型2" "模型3") ///
    addnotes("标准误在括号中" "*** p<0.01, ** p<0.05, * p<0.1")

*-------------------------------------------------------------------------------
*  6. 图形绘制
*-------------------------------------------------------------------------------

* 散点图加拟合线
twoway (scatter price weight, msize(small)) ///
       (lfit price weight, lwidth(medthick)), ///
    title("汽车价格与重量的关系") ///
    xtitle("重量 (磅)") ///
    ytitle("价格 (美元)") ///
    legend(label(1 "数据点") label(2 "拟合线")) ///
    scheme(s1mono) ///
    name(fig1, replace)

graph export "$figures/scatter_price_weight.png", replace width(1200)
graph export "$figures/scatter_price_weight.pdf", replace

* 系数图
coefplot m1 m2 m3, keep(mpg weight) ///
    xline(0, lcolor(red)) ///
    title("系数估计") ///
    scheme(s1mono) ///
    name(fig2, replace)

graph export "$figures/coefficient_plot.png", replace width(1200)

*-------------------------------------------------------------------------------
*  7. 保存结果
*-------------------------------------------------------------------------------

* 保存处理后的数据
save "$clean/analysis_data_processed.dta", replace

* 保存估计结果（用于后续处理）
estimates save "$results/estimates.ster", replace

*-------------------------------------------------------------------------------
*  8. 收尾
*-------------------------------------------------------------------------------

* 关闭日志
log close

* 显示结束时间
display "结束时间: $S_TIME $S_DATE"

* 退出（可选）
* exit, clear STATA

项目目录结构推荐

project_name/
├── README.md                    # 项目说明
├── LICENCE                      # 许可证
├── data/                        # 数据目录
│   ├── raw/                     # 原始数据（只读）
│   ├── clean/                   # 清洗后的数据
│   └── temp/                    # 临时数据
├── code/                        # 代码目录
│   ├── 01_data_cleaning.do      # 数据清洗
│   ├── 02_descriptive_stats.do  # 描述性统计
│   ├── 03_main_analysis.do      # 主要分析
│   └── 04_robustness.do         # 稳健性检验
├── results/                     # 结果目录
│   ├── logs/                    # 日志文件
│   ├── tables/                  # 表格
│   ├── figures/                 # 图形
│   └── intermediate/            # 中间结果
├── docs/                        # 文档
│   └── notes.md
└── paper/                       # 论文
    ├── manuscript.tex
    └── supplement.tex

注释标准

*-------------------------------------------------------------------------------
*  区块标题：使用这种格式分隔主要部分
*-------------------------------------------------------------------------------

* 单行注释：使用星号，简要说明

// 也可以使用双斜杠，但在 Stata 中不常见

/*
   多行注释：
   用于较长的说明文字
   可以跨越多行
*/

// TODO: 标记待完成的任务
// FIXME: 标记需要修复的问题
// NOTE: 重要提示信息

* 行内注释示例
sysuse auto, clear          // 加载汽车数据集
summarize price mpg weight  // 查看关键变量统计

* 复杂计算的解释
gen double price_per_mpg = price / mpg  // 计算每英里油耗成本
label var price_per_mpg "每加仑英里价格"

3.1.2 log using 日志文件记录方法

基础用法

* 简单日志
log using "analysis.log", replace text

* 带时间戳的日志
local datetime: di %tdCCYY-NN-DD date(c(current_date), "DMY")
log using "analysis_`datetime'.log", replace text

* 带时间和日期的完整日志
local logname "analysis_$S_DATE_$S_TIME.log"
local logname = subinstr("`logname'", ":", "-", .)
log using "`logname'", replace text

日志管理技巧

* 暂时关闭日志
log off

* 重新开启日志
log on

* 查看日志状态
log query

* 关闭日志
log close

封装的日志函数

* 保存为 start_log.ado
capture program drop start_log
program define start_log
    syntax using/, [replace append]

    * 创建日志目录
    mata: dirs = pathsplit("`using'")
    mata: logdir = pathjoin(dirs[1..cols(dirs)-1])
    cap mkdir "`logdir'"

    * 添加时间戳
    local datetime = subinstr("$S_DATE", " ", "_", .) + "_" + ///
                    subinstr("$S_TIME", ":", "-", .)
    local basename = subinstr("`using'", ".log", "", .)

    log using "`basename'_`datetime'.log", `replace' `append' text

    display "日志已启动: `basename'_`datetime'.log"
    display "开始时间: $S_TIME $S_DATE"
end

* 使用方法
start_log using "$logs/analysis"

3.1.3 结果输出路径规划与目录组织

全局路径管理

* 方法1: 使用 profile.do 设置全局根目录
* 在 profile.do 中定义:
global project_root "D:/research/my_project"

* 在各个 do-file 中使用:
global data    "$project_root/data"
global results "$project_root/results"

* 方法2: 使用相对路径（推荐用于Git项目）
* 假设 do-file 在 code/ 目录下
cd ..
global root = c(pwd)
global data   "$root/data"
global results "$root/results"
cd "`c(pwd)'/code"  // 返回原目录

* 方法3: 使用 project 命令（如果安装）
* ssc install project
project "my_project", do("code/analysis.do")

自动创建目录结构

* 定义目录列表
local dirs ///
    "$root/data/raw" ///
    "$root/data/clean" ///
    "$root/data/temp" ///
    "$root/results/logs" ///
    "$root/results/tables" ///
    "$root/results/figures" ///
    "$root/results/intermediate"

* 循环创建目录
foreach dir of local dirs {
    capture mkdir "`dir'"
    if _rc == 0 {
        display "创建目录: `dir'"
    }
}

文件命名约定

* 数据文件: [阶段]_[描述].dta
* - raw_: 原始数据
* - clean_: 清洗后数据
* - analysis_: 分析用数据

use "$data/raw/raw_survey.dta", clear
save "$data/clean/clean_survey.dta", replace
save "$data/clean/analysis_sample.dta", replace

* 结果文件: [类型]_[描述]_[日期].[后缀]
esttab ... using "$tables/reg_main_20260624.tex", replace
graph export "$figures/heterogeneity_urban.pdf", replace

* 日志文件: [do文件名]_[时间].log
log using "$logs/03_main_analysis_2026-06-24_1430.log", replace

3.2 常见报错与调试技巧

3.2.1 典型错误类型解析

文件相关错误

错误代码 错误信息 原因 解决方法
r(601) file … not found 文件路径错误 检查文件路径和拼写
r(602) file … could not be opened 权限问题 关闭其他程序中打开的文件
r(603) file … cannot be modified 文件只读 去除文件只读属性
r(608) file … already exists 文件已存在 使用 replace 选项或改名 
* 安全的文件操作
capture confirm file "$data/analysis.dta"
if _rc != 0 {
    display as error "错误: 数据文件不存在!"
    exit 601
}

* 检查目录是否存在
capture cd "$results"
if _rc != 0 {
    display as error "错误: 结果目录不存在!"
    exit
}

变量相关错误

错误代码 错误信息 原因 解决方法
r(111) variable … not found 变量名拼写错误 检查变量名和数据
r(109) type mismatch 变量类型错误 使用 destring/tostring 转换
r(101) variable … already defined 变量重复定义 使用 capture 或先 drop
r(198) invalid syntax 语法错误 检查命令语法 
* 安全的变量生成
capture confirm variable price_per_mpg
if _rc == 0 {
    drop price_per_mpg
}
gen price_per_mpg = price / mpg

* 或者使用 capture
capture drop price_per_mpg
gen price_per_mpg = price / mpg

* 检查变量类型
describe price
if r(type) == "str" {
    destring price, replace
}

数据相关错误

错误代码 错误信息 原因 解决方法
r(459) no observations 数据为空 检查数据加载
r(480) invalid observation number 行号越界 检查观测数
r(2000) no; data in memory would be lost 未保存数据 使用 clear, force 或先保存
r(198) invalid name 名称不合法 使用有效名称 
* 安全的数据操作
capture assert _N > 0
if _rc != 0 {
    display as error "错误: 数据为空!"
    exit 459
}

* 检查缺失值
count if missing(price, mpg, weight)
if r(N) > 0 {
    display as warning "警告: 存在 " r(N) " 个缺失值"
}

* 保存前检查
confirm variable price mpg weight
save "$data/analysis.dta", replace

3.2.2 调试策略与问题定位方法

逐步调试法

* 方法1: 设置断点
* 在关键位置添加 pause
display "检查点1: 数据已加载"
pause

* 方法2: 分段执行
* 将 do-file 分成多个部分，逐个测试

* 方法3: 使用 trace
set trace on
set tracedepth 2
* 执行问题代码...
set trace off

调试工具函数

* 保存为 debug_utils.ado

* 显示变量信息
capture program drop var_info
program define var_info
    syntax varlist

    foreach var of varlist `varlist' {
        display as text "--- `var' ---"
        capture describe `var'
        if _rc == 0 {
            local type: type `var'
            local label: variable label `var'
            local n_miss = missing(`var')

            display "  类型: `type'"
            display "  标签: `label'"
            display "  缺失: `n_miss'"
            summarize `var'
        }
        display ""
    }
end

* 使用方法: var_info price mpg weight

* 检查数据完整性
capture program drop data_check
program define data_check
    syntax [varlist]

    display "=== 数据完整性检查 ==="
    display "观测数: " _N
    display "变量数: " c(k)
    display ""

    if "`varlist'" == "" {
        local varlist _all
    }

    foreach var of varlist `varlist' {
        local n_miss = missing(`var')
        local pct_miss = `n_miss' / _N * 100

        if `n_miss' > 0 {
            display as error "`var': `n_miss' 缺失 (" %3.1f `pct_miss' "%)"
        }
        else {
            display as text "`var': 完整"
        }
    }
end

使用 capture 和 assert 进行防御性编程

* 示例: 稳健的数据分析流程

* 1. 设置参数
local depvar price
local covariates mpg weight length

* 2. 检查变量存在
foreach var of varlist `depvar' `covariates' {
    capture confirm variable `var'
    if _rc != 0 {
        display as error "错误: 变量 `var' 不存在!"
        exit 111
    }
}

* 3. 检查样本量
regress `depvar' `covariates'
capture assert e(N) > 50
if _rc != 0 {
    display as error "错误: 样本量不足!"
    exit 459
}

* 4. 保存结果
estimates store main_model
display "分析完成! 样本量: " e(N)

3.2.3 可复现性研究的最佳实践

版本控制与 Git

创建 .gitignore 文件：

# Stata 临时文件
*.log
*.smcl
*.dta
*.gph
*.ster

# 结果文件
results/
logs/
figures/

.DS_Store

初始化 Git 仓库：

# 在项目根目录执行
git init
git add README.md
git add code/*.do
git add .gitignore
git commit -m "Initial commit"

包版本管理

* 记录已安装包的版本
capture program drop save_packages
program define save_packages
    syntax using/

    tempname fh
    file open `fh' using "`using'", write replace

    file write `fh' "* Stata 包列表 - 生成时间: $S_DATE $S_TIME" _n
    file write `fh' "* 使用方法: do install_packages.do" _n _n

    * 获取所有用户安装的包
    local packages: dir "`c(sysdir_plus)'" dirs "*", respectcase

    foreach pkg of local packages {
        local pkgname = subinstr("`pkg'", "+", " ", .)
        file write `fh' `"""ssc install `pkgname', replace"""' _n
    }

    file close `fh'
    display "包列表已保存到: `using'"
end

* 使用方法
save_packages using "install_packages.do"

完整的可复现工作流示例

* master.do - 一键运行完整分析
*-------------------------------------------------------------------------------

* 记录开始时间
local start_time = c(current_time)
local start_date = c(current_date)
display "分析开始: `start_date' `start_time'"

* 设置随机种子（确保可复现）
set seed 20260624

* 记录 Stata 版本
display "Stata 版本: `c(stata_version)'"
display "操作系统: `c(os)' `c(osdtl)'"

* 检查必需的包
local required_packages ftools gtools reghdfe estout coefplot
foreach pkg of local required_packages {
    capture ssc describe `pkg'
    if _rc != 0 {
        display as error "需要安装: `pkg'"
        ssc install `pkg', replace
    }
}

* 运行各分析步骤
do "code/01_data_cleaning.do"
do "code/02_descriptive_stats.do"
do "code/03_main_analysis.do"
do "code/04_robustness.do"

* 生成最终报告
* do "code/make_report.do"

* 计算运行时间
local end_time = c(current_time)
display "分析完成: `end_date' `end_time'"
display "运行结束"

Makefile 示例（高级用户）

创建 Makefile：

# Stata 分析工作流 Makefile

STATA = "C:/Program Files/Stata18/StataMP-64.exe"
DOFLAGS = -e do

.PHONY: all clean data analysis figures

all: analysis figures

data: data/clean/analysis_data.dta

data/clean/analysis_data.dta: code/01_data_cleaning.do data/raw/raw_data.dta
    $(STATA) $(DOFLAGS) $<

analysis: results/tables/regression_results.tex

results/tables/regression_results.tex: code/03_main_analysis.do data/clean/analysis_data.dta
    $(STATA) $(DOFLAGS) $<

figures: results/figures/main_figure.pdf

results/figures/main_figure.pdf: code/03_main_analysis.do
    $(STATA) $(DOFLAGS) $<

clean:
    rm -f results/logs/*.log
    rm -f data/temp/*.dta

cleanall: clean
    rm -f data/clean/*.dta
    rm -f results/tables/*.tex
    rm -f results/figures/*.pdf

使用方法：

# 运行所有分析
make all

# 只重新生成数据
make data

# 清理临时文件
make clean
