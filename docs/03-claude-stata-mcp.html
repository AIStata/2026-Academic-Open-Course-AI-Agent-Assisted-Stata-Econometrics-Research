4.1 Stata MCP 配置与原理

4.1.1 Model Context Protocol 简介

什么是 MCP？

模型上下文协议 (Model Context Protocol, MCP) 是 Anthropic 开发的开放式协议，用于在 AI 助手和外部工具之间建立标准化连接。它让 Claude 能够：

  • 直接访问本地文件系统
  • 读取和写入文件
  • 执行命令行工具
  • 与数据库交互
  • 调用特定领域的 API

传统工作流 vs MCP 工作流

传统方式：复制粘贴

1. 在 Stata 中查看数据
2. 复制结果到 Claude
3. Claude 给出代码
4. 复制代码回 Stata
5. 运行代码查看结果
6. 重复...

MCP 方式：直接执行

1. Claude 通过 MCP 直接连接 Stata
2. Claude 读取数据、分析结果
3. Claude 直接在 Stata 中执行代码
4. 实时获取结果并迭代分析

4.1.2 从”复制粘贴”到”直接执行”的工作流转变

传统工作流的痛点

flowchart TD
    A[Stata 分析] --> B{遇到问题}
    B -->|复制错误| C[打开 Claude]
    C -->|粘贴问题| D[等待回复]
    D -->|获取代码| E[复制回 Stata]
    E --> F[运行代码]
    F --> G{是否解决?}
    G -->|否| B
    G -->|是| H[继续分析]

MCP 工作流的优势

flowchart TD
    A[开始分析] --> B[Claude + Stata MCP]
    B --> C{Claude 直接操作}
    C -->|读取数据| D[分析问题]
    C -->|执行代码| E[获取结果]
    D --> F[提供解决方案]
    E --> F
    F --> G{满意?}
    G -->|是| H[完成]
    G -->|否| C

4.2.2 AI 代码解释与逻辑拆解

理解复杂计量方法

示例：理解 DID 模型

用户: 请解释以下 DID 代码的逻辑，并用简单的语言说明

gen treat = (country == "treatment")
gen post = (year >= 2020)
gen did = treat * post

reg outcome treat post did, robust

Claude 的解释：

这段代码实现了双重差分（DID）模型，逻辑如下：

1. **分组变量 (`treat`)**
   - 处理组 = 1（政策实施地区）
   - 对照组 = 0（其他地区）

2. **时间变量 (`post`)**
   - 政策后 = 1（2020年及以后）
   - 政策前 = 0（2020年以前）

3. **交互项 (`did`)**
   - 这是关键变量！系数表示政策效果
   - 即：处理组在政策后的变化 减去 对照组的变化

4. **回归方程**
   outcome = β₀ + β₁treat + β₂post + β₃did + ε
   - β₁: 组间固定差异
   - β₂: 时间趋势
   - β₃: DID 估计量（政策效果）✭

用通俗的话说：我们比较处理组在政策前后的变化，
同时减去对照组在同一时期的变化，这样就能分离出
政策的净效果，排除其他同时期发生的因素影响。

需要我生成一个可视化的例子吗？

代码优化建议

原代码：

gen resid = y - yhat

4.2.3 代码改写与优化建议

示例1：从低效到高效

原始代码：

use data.dta, clear

gen lnprice = log(price)
gen lnincome = log(income)

reg lnprice lnincome if year == 2010
estimates store m1

reg lnprice lnincome if year == 2011
estimates store m2

reg lnprice lnincome if year == 2012
estimates store m3

esttab m1 m2 m3 using results.tex, replace

Claude 优化后：

use data.dta, clear

* 生成变量
gen lnprice = log(price)
gen lnincome = log(income)

* 使用循环简化代码
estimates clear
forvalues year = 2010/2012 {
    reg lnprice lnincome if year == `year', robust
    estimates store m`year'
}

* 输出带年份标签的表格
esttab m2010 m2011 m2012 using "$tables/yearly_reg.tex", replace ///
    se star(* 0.1 ** 0.05 *** 0.01) ///
    label mtitles("2010" "2011" "2012") ///
    addnotes("稳健标准误在括号中")

* 或者更高效的方法：使用交互项
reg lnprice lnincome i.year i.year#c.lnincome, robust
testparm i.year#c.lnincome

示例2：添加错误处理

Claude 生成的健壮代码：

*-------------------------------------------------------------------------------
*  安全的数据分析脚本
*-------------------------------------------------------------------------------

capture log close
set more off

* 参数设置
local datafile "$data/analysis_data.dta"
local depvar lwage
local indvars educ exper tenure

* 检查文件
capture confirm file "`datafile'"
if _rc != 0 {
    display as error "错误: 数据文件不存在！"
    exit 601
}

* 加载数据
use "`datafile'", clear

* 检查变量
foreach var of varlist `depvar' `indvars' {
    capture confirm variable `var'
    if _rc != 0 {
        display as error "错误: 变量 `var' 不存在！"
        exit 111
    }
}

* 检查缺失值
foreach var of varlist `depvar' `indvars' {
    count if missing(`var')
    if r(N) > 0 {
        display as warning "警告: `var' 有 " r(N) " 个缺失值"
    }
}

* 回归分析
regress `depvar' `indvars', robust

* 检查样本量
if e(N) < 50 {
    display as error "错误: 样本量不足！"
    exit 459
}

* 保存结果
estimates store main_model
esttab main_model using "$tables/results.tex", replace ///
    se star(* 0.1 ** 0.05 *** 0.01)

display "分析成功完成！"

#### 场景2：研究设计反馈

用户: 我准备这样分析这个问题，帮我看看有什么问题：

  1. 用 OLS 回归 Y = βX + ε
  2. 如果显著就得出结论
  3. 再加入一些控制变量看看结果是否变化

**Claude 的反馈：**

你的设计有几个需要改进的地方：

4.6 潜在问题：

  1. 遗漏变量偏误
    • 只加控制变量不够，需要考虑选择哪些控制变量
    • 建议画因果图（DAG）确定需要控制的变量
  2. 缺乏识别策略
    • OLS 可能存在内生性
    • 需要考虑：DID、IV、RDD 等识别方法
  3. 稳健性不足
    • 应该报告多种设定的结果
    • 做安慰剂检验、子样本分析等

4.7 改进建议：

* 1. 先做描述性统计和相关性分析
estpost correlate y x controls
esttab using "$tables/corr.tex", replace

* 2. 逐步加入控制变量
estimates clear
reg y x, robust
estimates store m1
reg y x controls1, robust
estimates store m2
reg y x controls1 controls2, robust
estimates store m3

* 3. 考虑识别策略
* DID: reg y i.treat i.post i.treat#i.post controls, robust
* IV: ivregress 2sls y (x = z) controls, robust

* 4. 稳健性检验
* 子样本、安慰剂、不同标准误设定

estimates store se_cluster

* 2. 子样本分析
xtreg gdp_growth trade ln_investment ln_education i.year if year >= 2000, fe robust
estimates store sub_recent

xtreg gdp_growth trade ln_investment ln_education i.year if gdp_per_capita > 10000, fe robust
estimates store sub_rich

* 3. 异质性 - 按收入水平
xtreg gdp_growth c.trade##i.income_group ln_investment ln_education i.year, fe robust
estimates store hetero_income

* 4. 动态面板（系统GMM）
* ssc install xtabond2
xtabond2 gdp_growth L.gdp_growth trade ln_investment ln_education i.year, ///
    gmm(L.gdp_growth trade, lag(2 4)) iv(ln_investment ln_education i.year) ///
    twostep robust small
estimates store gmm

* 输出稳健性检验结果
esttab se_robust se_cluster sub_recent sub_rich ///
    using "$tables/robustness.tex", replace ///
    se star(* 0.1 ** 0.05 *** 0.01) ///
    keep(trade) label ///
    mtitles("Robust" "Cluster" "Recent" "Rich")
