2.1 Stata 基础设置

2.1.1 Profile 文件设置

Stata 的 profile.do 文件是 Stata 启动时自动执行的脚本文件，用于配置个性化工作环境。

查找 Profile 文件位置

Profile 文件应放置在 Stata 的启动目录下。可以通过以下命令查看当前目录：

pwd

创建 Profile 文件

在 Stata 中输入以下命令创建或编辑 profile.do：

doedit profile.do

推荐的 Profile 配置

*-------------------------------------------------------------------------------
*  Stata Profile 配置文件
*  数量经济学 - 2026学术公开课
*-------------------------------------------------------------------------------

* 基本设置
set more off, permanently
set linesize 120, permanently

set scrollbufsize 200000, permanently

set varabbrev off, permanently

2.1.2 PLUS 目录配置与外部命令管理

理解 Stata 的 ado 目录结构

Stata 使用多个目录来存储 ado 文件：

BASE/      Stata 官方基础命令
PLUS/      用户安装的外部命令
PERSONAL/  用户个人编写的命令
SITE/      站点级命令
OLDPLACE/  旧版兼容目录

配置 PLUS 目录

查看当前 PLUS 目录：

sysdir list

设置 PLUS 目录（示例）：

* Windows 系统
sysdir set PLUS "C:/Stata/ado/plus"
sysdir set PERSONAL "C:/Stata/ado/personal"

* macOS 系统
sysdir set PLUS "~/Documents/Stata/ado/plus"
sysdir set PERSONAL "~/Documents/Stata/ado/personal"

* Linux 系统
sysdir set PLUS "~/stata/ado/plus"
sysdir set PERSONAL "~/stata/ado/personal"

将永久设置写入 profile.do：

doedit profile.do

外部命令安装与管理

常用外部命令安装：

* 基础工具包
ssc install ftools, replace
ssc install gtools, replace
ssc install reghdfe, replace
ssc install estout, replace
ssc install coefplot, replace

* 数据处理
ssc install tidy, replace
ssc install gdata, replace

* 绘图扩展
ssc install grstyle, replace
ssc install palettes, replace
ssc install colrspace, replace

* 因果推断
ssc install did_multiplegt, replace
ssc install did_imputation, replace
ssc install csdid, replace
ssc install drdid, replace

* 机器学习
ssc install rforest, replace
ssc install lassopack, replace
ssc install pdslasso, replace

检查已安装的命令：

ado dir

更新已安装的命令：

ado update, update

2.2 VS Code 中的 Stata 开发环境

2.2.1 Stata Enhanced 扩展安装与配置

安装扩展

  1. 打开 VS Code
  2. 点击左侧扩展图标（或按 Ctrl+Shift+X）
  3. 搜索 “Stata Enhanced”
  4. 点击安装

推荐安装的扩展：

扩展名称 用途
Stata Enhanced Stata 语法高亮与代码执行
Stata LSP Stata 语言服务器
Stata Jupyter Kernel Jupyter 中运行 Stata
GitHub Copilot AI 代码补全
Code Spell Checker 拼写检查
Markdown All in One Markdown 编辑
Quarto Quarto 文档编辑
Git Graph Git 可视化

配置 Stata Enhanced

在 VS Code 设置中（Ctrl+,）搜索 “Stata”，进行以下配置：

{
  "stata.stataPath": "C:/Program Files/Stata18/StataMP-64.exe",
  "stata.version": 18,
  "stata.allowShortcuts": true,
  "stata.executeDelay": 100,
  "stata.includeComments": true
}

Windows 用户配置示例：

{
  "stata.stataPath": "C:/Program Files/Stata18/StataMP-64.exe",
  "stata.version": 18
}

macOS 用户配置示例：

{
  "stata.stataPath": "/Applications/Stata/StataMP.app",
  "stata.version": 18
}

Linux 用户配置示例：

{
  "stata.stataPath": "/usr/local/stata18/stata-mp",
  "stata.version": 18
}

2.2.2 语法高亮、代码折叠、自动补全功能

语法高亮配置

Stata Enhanced 提供了完善的语法高亮，支持：

  • 命令关键字高亮
  • 宏和函数识别
  • 注释区分
  • 字符串高亮
  • 数字高亮

自定义颜色主题示例（在 settings.json 中）：

{
  "editor.tokenColorCustomizations": {
    "textMateRules": [
      {
        "scope": "keyword.control.stata",
        "settings": {
          "foreground": "#569CD6"
        }
      },
      {
        "scope": "string.quoted.double.stata",
        "settings": {
          "foreground": "#CE9178"
        }
      }
    ]
  }
}

代码折叠设置

在 settings.json 中配置代码折叠：

{
  "editor.folding": true,
  "editor.foldingStrategy": "indentation",
  "editor.foldingHighlight": true,
  "editor.showFoldingControls": "always"
}

Stata 代码中可使用特殊标记来定义可折叠区域：

*-------------------------------------------------------------------------------
* 区域1: 数据加载 {{{
*-------------------------------------------------------------------------------
sysuse auto, clear
describe
summarize
* }}}

*-------------------------------------------------------------------------------
* 区域2: 回归分析 {{{
*-------------------------------------------------------------------------------
regress price mpg weight length
estimates store model1
* }}}

使用 {{{ 和 }}} 标记可以创建自定义折叠区域。

自动补全功能

VS Code 为 Stata 提供以下自动补全：

  1. 命令补全：输入命令前缀，按 Tab 或 Ctrl+Space
  2. 变量名补全：在代码中引用变量时自动提示
  3. 选项补全：命令选项自动提示
  4. 路径补全：文件路径自动完成

配置自动补全行为：

{
  "editor.quickSuggestions": {
    "other": true,
    "comments": false,
    "strings": true
  },
  "editor.tabCompletion": "on",
  "editor.wordBasedSuggestions": true,
  "editor.parameterHints.enabled": true
}

2.2.3 Do-file 编辑器配置与运行方式设置

文件关联配置

确保 .do 和 .ado 文件与 Stata Enhanced 关联：

{
  "files.associations": {
    "*.do": "stata",
    "*.ado": "stata",
    "*.mata": "stata"
  }
}

代码片段（Snippets）

创建 Stata 代码片段，提高编码效率。在 VS Code 中：

  1. 按 Ctrl+Shift+P
  2. 输入 “Configure User Snippets”
  3. 选择 “stata.json”

添加常用代码片段：

{
  "Stata Header": {
    "prefix": "header",
    "body": [
      "*-------------------------------------------------------------------------------",
      "*  ${1:文件名}.do",
      "*  目的: ${2:描述}",
      "*  作者: ${3:你的名字}",
      "*  日期: ${CURRENT_YEAR}-${CURRENT_MONTH}-${CURRENT_DATE}",
      "*-------------------------------------------------------------------------------",
      "",
      "clear all",
      "set more off",
      "set linesize 120",
      "",
      "* 设置路径",
      "global path \"${4:路径}\"",
      "cd \"$path\"",
      "",
      "* 开始日志",
      "log using \"${1:文件名}.log\", replace text",
      "",
      "$0",
      "",
      "* 关闭日志",
      "log close",
      "exit",
      "*/"
    ],
    "description": "标准 Stata Do-file 头部"
  },

  "Regression Table": {
    "prefix": "esttab",
    "body": [
      "estimates clear",
      "regress ${1:y} ${2:x1 x2}",
      "estimates store m1",
      "",
      "regress ${1:y} ${2:x1 x2} ${3:x3 x4}",
      "estimates store m2",
      "",
      "esttab m1 m2 using \"${4:results}.tex\", replace",
      "esttab m1 m2, se star(* 0.1 ** 0.05 *** 0.01)"
    ],
    "description": "esttab 回归表格模板"
  },

  "Graph Export": {
    "prefix": "graphexport",
    "body": [
      "twoway (scatter ${1:y} ${2:x}) (lfit ${1:y} ${2:x}),",
      "    title(\"${3:标题}\")",
      "    subtitle(\"${4:副标题}\")",
      "    xtitle(\"${5:X轴标签}\")",
      "    ytitle(\"${6:Y轴标签}\")",
      "    legend(label(1 \"数据点\") label(2 \"拟合线\"))",
      "    scheme(s1mono)",
      "",
      "graph export \"${7:output}.png\", replace width(1200)",
      "graph export \"${7:output}.pdf\", replace"
    ],
    "description": "图形导出模板"
  }
}

运行 Do-file 的方法

  1. 运行整个文件：
    • 按 Ctrl+Shift+Enter 运行整个 do-file
    • 或右键选择 “Stata: Execute Do File”
  2. 运行选中代码：
    • 选中代码片段
    • 按 Ctrl+Enter 运行选中部分
  3. 运行当前行：
    • 光标放在某一行
    • 按 Ctrl+Enter 运行当前行

配置快捷键：

{
  "key": "ctrl+enter",
  "command": "stata.runSelection",
  "when": "editorLangId == stata"
},
{
  "key": "ctrl+shift+enter",
  "command": "stata.runAll",
  "when": "editorLangId == stata"
},
{
  "key": "ctrl+shift+c",
  "command": "stata.showResult",
  "when": "editorLangId == stata"
}

编辑器配置推荐

在 .vscode/settings.json 中创建项目级设置：

{
  // Stata 相关
  "stata.stataPath": "C:/Program Files/Stata18/StataMP-64.exe",
  "stata.version": 18,

  // 编辑器
  "editor.tabSize": 4,
  "editor.insertSpaces": true,
  "editor.rulers": [80, 100, 120],
  "editor.wordWrap": "on",
  "editor.minimap.enabled": true,

  // 自动保存
  "files.autoSave": "afterDelay",
  "files.autoSaveDelay": 1000,

  // 格式化
  "editor.formatOnSave": false,
  "editor.formatOnPaste": true,

  // 括号配对
  "editor.guides.bracketPairs": true,
  "editor.bracketPairColorization.enabled": true,

  // Git
  "git.enableSmartCommit": true,
  "git.autofetch": true
}

同时创建 .vscode/extensions.json 推荐扩展：

{
  "recommendations": [
    "kylebarron.stata-enhanced",
    "yzhang.markdown-all-in-one",
    "quarto.quarto",
    "GitHub.copilot",
    "eamodio.gitlens"
  ]
}
